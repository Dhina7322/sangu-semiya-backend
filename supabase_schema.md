# Supabase SQL Schema Configuration

Follow these steps to set up your PostgreSQL database in Supabase:

### 1. Execute SQL in Supabase Dashboard
Go to your **Supabase Dashboard** -> **SQL Editor** -> **New Query** and run the following commands to create your tables.

```sql
-- 1. Create Users Table
CREATE TABLE IF NOT EXISTS users (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  email TEXT UNIQUE NOT NULL,
  password TEXT NOT NULL,
  role TEXT DEFAULT 'Sub-Admin',
  created_at TIMESTAMPTZ DEFAULT now()
);

-- 2. Create Products Table
CREATE TABLE IF NOT EXISTS products (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  name TEXT NOT NULL,
  sku TEXT UNIQUE,
  description TEXT,
  images TEXT[] DEFAULT '{}',
  pack_size TEXT,
  amazon_link TEXT,
  price TEXT,
  category TEXT DEFAULT 'General',
  variants TEXT,
  status TEXT DEFAULT 'Active',
  featured BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- 3. Create Enquiries Table
CREATE TABLE IF NOT EXISTS enquiries (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  name TEXT NOT NULL,
  phone TEXT NOT NULL,
  email TEXT,
  product TEXT NOT NULL,
  quantity TEXT NOT NULL,
  message TEXT,
  status TEXT DEFAULT 'Pending',
  created_at TIMESTAMPTZ DEFAULT now()
);

-- 4. Create Homepage Table
CREATE TABLE IF NOT EXISTS homepage (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  hero_banner JSONB DEFAULT '{"message": "Sangu Brand Semiya", "subMessage": "Healthy, Delicious & Quick to Cook", "backgroundImage": ""}',
  why_choose_us JSONB DEFAULT '[]',
  production_steps JSONB DEFAULT '[]',
  about_text TEXT DEFAULT '',
  contact_details JSONB DEFAULT '{"phone": "", "email": "", "address": "", "whatsapp": ""}',
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Insert initial empty homepage row
INSERT INTO homepage (id) VALUES (1) ON CONFLICT (id) DO NOTHING;
```

### 2. Configure Storage
1. Go to **Storage** in the Supabase Dashboard.
2. Create a new bucket named **`product-images`**.
3. Set the bucket to **Public**.
4. In **Policies**, ensure that `anon` and `authenticated` roles can **SELECT**, **INSERT**, and **UPDATE** objects (or use the helper "Allow public access to all objects").

### 3. Verify `.env`
Ensure your `.env` file contains:
- `SUPABASE_URL`: Your project URL
- `SUPABASE_KEY`: Your project key (Service Role or Anon key with appropriate policies)
- `JWT_SECRET`: A secure random string
